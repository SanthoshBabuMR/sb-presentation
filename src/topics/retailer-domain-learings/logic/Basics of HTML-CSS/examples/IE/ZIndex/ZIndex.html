What Is

The z-index property specifies the stack order of an element.

In General, greater the z-index value of a HTML Element, they are always formatted in front of HTML Elements with lower z-index levels, in the same stacking context.

Note: In CSSÂ 2.1, each box has a position in three dimensions. In addition to their horizontal and vertical positions, boxes lie along a "z-axis" and are formatted one on top of the other. Z-axis positions are particularly relevant when boxes overlap visually - http://www.w3.org/TR/CSS21/visuren.html#z-index

It is important to note that z-index works only with positioned elements [absolute, relative].

Understanding z-index

When we require a HTML Element to be formatted in front of other HTML Elements, we simply assume, specifying greater z-index value, would address our need.

But, often this is not the case. When a z-index is specified to a positioned element, a new stacking context is created.

For example,

<body>

<div id="Orange" style="position:relative; z-index:2">

</div>

<div id="Blue" style="position:relative; z-index:1>

</div>

</body>

would create two stacking context for Orange and Blue respectively. As Blue has lower z-index, it is rendered behind Orange.

In other words, Orange appears closer to screen than Blue, as shown below,

Example

Lets us start with two Container DIV's - topContainer and bottomContainer. The topContainer contains Child DIV.

Static Positioning and Default Stacking Context

All the Elements[topContainer, bottomContainer and Child-DIV] are static postioned, hence we have only the default stacking context, which is the normal flow of elements in a web page.

position: staic;

Relative Positioning and Stacking Context

topContainer and bottomContainer, are relatively positioned with z-index values of 1 and 2 respectively.

This creates two stacking context, and the Child DIV has same stacking context as its parent. In this case, both the topContainer and ChildDiv appear behind bottomContainer, as bottomContainer has higher stacking context.

Relative/Absolute Positioning and Default Stacking Context

topContainer, bottomContainer are relatively positioned while Child DIV is absolutely positioned to overlap with bottomContainer.

In this case, as no z-index is specified, only default stacking context exist, yet Child DIV is hidden behind bottomContainer.

With Default Stacking Context, the order in which elements are coded takes precedence; The last element in the web page takes priority over rest of the elements[note that all elements here are positioned]; So, if we were to position topContainer on top of bottomContainer, then topContainer would be hidden behind bottomContainer.

Z-Index Rules

When it comes to z-index, the following applies

  1. Positioned elements [absolute/relative] take priority over static elements. [note that priority here, refers to being able to come "in front of"]
  2. If there are multiple positioned elements, then HTML Elements at the last, take priority over the rest (i.e) if we have "n" elements in order, then nth element takes priority over the remaining (n-1) elements
    Eg:
    • <body>
    • <div style="position:relative">
    • <!-- code -->
    • </div>
    • <a style="position:relative">
    • <!-- code -->
    • </a>
    • <ul>
    • <!-- code -->
    • </ul>
    • <p style="position:relative">
    • <!-- code -->
    • </p>
    • </body>
    • Here, <p> element and its descendants, have greater priority over <ul>, <a> and <div> elements
      <a> element and its descendants, have greater priority over <ul> and <div> elements
      <div>element and its descendants, have greater priority only over <ul> element
      <ul> being non-positioned, is in default stacking context.
  3. If multiple elements are positioned and given z-index, then stacking context is created accordingly. The positioned element with greatest z-index takes priority and comes in front of other elements and so on.
  4. If multiple elements are positioned with z-index values, then the Child Elements that belong to highest stacking context [positioned element with highest z-index] is displayed in front of other elements.

    Now, with in the Positioned Element, the Child elements follow 1,2 & 3 rules accordingly.

IE Issue

Assuming topContainer and bottomContainer are relatively positioned, the Child-DIV in topContainer is absolutely positioned with a z-index value of 500.

Since topContainer, bottomContainer and Child-DIV of topContainer are positioned, three stacking context are created due to the z-index values.

Though Child DIV z-index value is greater than other elements, it doesn't get displayed on top. This is because, it exists in the context of topContainer, whose z-index is lower than bottomContainer. Hence, the Child DIV or any Child from topContainer, will be hidden behind bottomContainer.

There are two ways to get the display right, and the methods are shown below.

TopContainer
position: relative;
z-index:1;
position: absolute;
z-index: 500;
This is Child-DIV from TopContainer
BottomContainer
position: relative;
z-index: 2;

Solution

In order to get the Child DIV on top of bottomContainer, we have two methods

Method 1:

we increase the z-index value of its parent(topContainer). Now any stacking context created within topContainer will come in front of bottomContainer and hence, Child DIV comes in front.

TopContainer
position: relative;
z-index:3
position: absolute;
z-index: 500;
This is Child-DIV from TopContainer
BottomContainer
position: relative;
z-index: 2;

Method 2:

[Note: This Method Doesn't Work in IE6 and IE7 and is the reason for inconsistent display across browsers; Follow Method 1 for Cross-Browser Consistency]

Remove the z-index value of the Child DIV's parent element and specify Child DIV with a higher z-index value than bottomContainer. (since, Child DIV already has a greater z-index value, we don't need to change it)

Note: If you view the below code in IE6 or IE7, the Child DIV will be hidden below bottomContainer.

TopContainer
position: relative;
position: absolute;
z-index: 500;
This is Child-DIV from TopContainer
BottomContainer
position: relative;
z-index: 2;

Why IE is Often the Culprit

We often find firefox or other browsers, to treat z-index correctly, while IE ignores it. This is often because Method 2 is employed to get the required layout style. Follow Method 1 and the layout will render as required.

Further, websites contains several layers of DIV's positioned absolutely or relatively, with different z-index values and sometimes, javascript is used to render layout dynamically, causing even more chaos and difficultly in pinpointing the cause of issue. Tools such as Firebug and Developer Tools could be of least useful, unless we understand how things function.

It is essential to maintian, a hierarchy of Elements regarding what comes on top over the other element. Tracking the Elements and their respective z-index also helps to design for future enhancment. But more than often, no such logs are maintained and we find needing to write hacks or IE specific css to address the issue.
