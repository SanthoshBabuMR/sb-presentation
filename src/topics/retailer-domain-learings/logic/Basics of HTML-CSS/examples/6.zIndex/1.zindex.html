zIndex

Scenario 1: Non-Positioned Elements overlapping

when two non-positioned(p1, p2) elements overlap, the latter element p2 would overlap on top of p1

Container p1 [p:static;]
Container p2 [p:static;]

Scenario 2: Postioned and Non-Positioned Elements overlapping

when a positioned element (p1) overlaps a non-positioned element (p2), then element p1 would come on top.

Positioned elements take priority over non-positioned elements

Container p1 [p:relative;]
Container p2 [p:static;]

Scenario 3: Positioned Elements overlapping with zIndex set

when a two positioned element (p1=500,p2=1000) overlaps, then the positioned element with highest zIndex(p2) value comes on top.

Positioned elements take priority over non-positioned elements

Container p1 [p:relative; z:500]
Container p2 [p:relative; z:1000]

Scenario 4: Positioned Elements with Childrens

when a two positioned element (p1=500,p2=1000) overlaps, then the positioned element with highest zIndex(p2) value comes on top.
Therefore all the children(s) contained within the p2 comes on top of p1 and its children(s)

Positioned elements take priority over non-positioned elements

Container p1 [p:relative; z:500]
P1 - Child [p:absolute; z:10000]
Container p2 [p:relative; z:1000]
P2 - Child [p:absolute; z:100]

Scenario 5: IE Issue

Internet Explorer versions 6 and 7 incorrectly reset the stacking context in relation to the nearest positioned ancestor element.

Container p1 [p:relative; z:9999]
Container p2 [p:relative; z:1]

IE6/7 will incorrectly display Container p2 on top of Container p1.
This is because, Container p2 is with in a DIV whose position is set to relative.
As a solution to this,
1. we need to remove the position property from Container p2's parent DIV.
2. (Or) set a Z-Index value to Container p1's Parent such that its greater than Container p2

The DropDown Menu in IE6/7 will be displayed behind the promo container div.
