rem

refers to font-size of the root element;

The advantage of rem is that, it avoids the compounding issue of font-size seen with em.

The approach would be setting a base font value to the body element, as shown below,

rem : setting font-size to body tag body { font-size: 14px; }

Now, we could define the font-size for the rest of the web elements with reference to the body element rather than the elements immediate parent container.

So, a paragraph tag with 16px, could be defined as,

setting font-size using rem p { font-size: 1.14285 rem;/* 16px /14px */ }

The clear advantage is that, we have a single point of reference for all the font-size we declare in the document, making future updates/enhancement much easier to deal with.

box-sizing

With box-sizing, W3C brings IE6 box-model into standards.

In a net shell, box-sizing, alters the way in which browser renders borders and padding of an HTML elements

The standard W3C box-model definition ,

Width of an element = border-left + padding-left + width defined + padding-right + border-right

Height of an element = border-top + padding-top + height defined + padding-bottom + border- bottom

So, if we wanted a div of width 100px, with a 1px border along its sides, then we end with a div of width 102px;

This becomes even worse when setting width in percentages; typically a container with a width of 100% is not supposed to have any borders or paddings, as it would break the layout.

What if we need to set an explicit width, and have borders, paddings but ensure that it is contained with the width specified? The solution is to use CSS3 box-sizing property.

In the above case, the solution would be to set the div with a css3 'box-sizing' property to 'border-box'.

box-sizing is about switching between the W3C Box model and ie6 box model.

box-sizing supports 2 valeus:

content-box:
retains the traditional W3C Box Model; borders and paddings are outside the width of an HTML Element.
border-box:
uses IE6 box model. ; borders and paddings are contained within the width of an HTML Element.

box-sizing could be an excellent choice of option for responsive design where we set values with a combination of percentages and pixels

flexbox

W3C specification:
In the flexbox layout model, the children of a flexbox can be laid out in any direction, and can 'flex' their sizes, either growing to fill unused space or shrinking to avoid overflowing the parent. Both horizontal and vertical alignment of the children can be easily manipulated.

Value: display: flex | inline-flex

Nesting of these boxes (horizontal inside vertical, or vertical inside horizontal) can be used to build layouts in two dimensions.

flex boxes helps in achieving equal-height columns by default and flex boxes could be helpful in adjusting layouts as we move from small screen to wide screen devices.

flexbox:
An element whose computed 'display' is either 'flex' or 'inline-flex' is a flexbox, and establishes a new flexbox formatting context for its contents.
flexbox item:
Children of a flexbox are called flexbox items

The list of flexbox properties,

flex-direction:
The 'flex-direction' property specifies how flexbox items are placed in the flexbox;
In other words, it specifies if the flexbox items must be laid horizontally or vertically within the flexbox.

Value: row | row-reverse | column | column-reverse

flex-wrap:
The 'flex-wrap' property controls whether the flexbox is single-line or multi-line, and the direction of the cross-axis, which determines the direction new lines are stacked in.

Value: nowrap | wrap | wrap-reverse

flex-flow:
The 'flex-flow' property is shorthand for setting the 'flex-direction' and 'flex-wrap' properties together.

Value: <'flex-direction'> || <'flex-wrap'>

flex-order:
Flexbox items are, by default, displayed and laid out in the same order as they appear in the source document. The 'flex-order' property may be used to change this ordering.

Value: <number>

flexbox will lay out its content starting from the lowest flex-order value to the highest. Items with the same flex-order value are laid out in the order they appear in the source document.

flex:
The defining aspect of flexbox layout is the ability to make the flexbox items 'flex', altering their width or height to fill the available space. This is done with the 'flex' property. A flexbox distributes free space to its items proportional to their positive flexibility, or shrinks them to prevent overflow proportional to their negative flexibility.

Value: none | [ <number>{1,2} || <'width'> ]

flex-pack:
The 'flex-pack' property aligns flexbox items along the main axis[left to right] of the current line of the flexbox.

Value: start | end | center | justify | distribute

flex-align:
The 'flex-align' property aligns flexbox items along the cross axis[top to bottom] of the current line of the flexbox.
'flex-align' sets the default alignment for all of the flexbox's items, including anonymous flexbox items.

Value: start | end | center | baseline | stretch

flex-item-align:
'flex-item-align' allows to override the default alignment for individual flexbox items, set using flex-align.

Value: auto | start | end | center | baseline | stretch

flex-line-pack:
The 'flex-line-pack' property aligns a flexbox's lines within the flexbox when there is extra space in the cross-axis, similar to how 'flex-pack' aligns individual items within the main-axis

Value: start | end | center | justify | distribute | stretch

CSS3 Background

CSS3 Multiple background-Image

CSS3 allows specifying multiple background images for box elements, using nothing more than a simple comma-separated list of background

CSS3 - Multiple Background .roundedcontainer{ background-image: url(tpleft.png), url(tpright.png), url(btleft.png), url(btright.png); background-position: left top, right top, left bottom, right bottom; background-repeat: no-repeat, no-repeat, no-repeat, no-repeat; background-color: rgb(243,78,156); }

CSS3 Multiple background-Size

Specifies the size of a background image

background-size: length|percentage|cover|contain;

contain:
ensures that background-image is fully rendered within the container, if the size of the container is less than the size of the background-image.
cover:
ensures that background-image is fully rendered within the container, if the size of the container is greater than the size of the background-image.
