Responsive Web Design is just a different approach to delivering information across different devices.

Moreover, it is an emerging practice and there are many challenges facing it, especially with that of delivering the appropriate images depending on the device accessing it.

There's quite a bit of people who doesn't support the idea of RWD; especially because, the needs of mobile users differs from that of desktop users.
http://blog.cloudfour.com/css-media-query-for-mobile-is-fools-gold/

Even if we had to work on designing two separate versions of website; one for mobile users and the other for desktop users; following the rules of RWD helps to create much better design.

The idea would be to have a mobile version, such that it can adapt to screen resolutions from 320px - 600px; and have the desktop version, considering resolution ranges of 768px - 1360px.

This way, we can overcome the issues we are currently faced with a single responsive design, i.e. image optimization, mobile user context, http requests and additional coding to suit both mobile and desktop versions. The downside to this approach would be maintaining the 2 websites.

The future of web designing clearly depends on a responsive design as devices being introduced come in different sizes and screen resolution.

As RWD matures, we should be capable of overcoming the challenges we face.
