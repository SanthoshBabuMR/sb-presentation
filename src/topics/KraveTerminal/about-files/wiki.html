Layout

KraveUpon.com is a 3 column layout - leftsidebar, maincontent, rightsidebar, with a header and footer.

As the leftsidebar is not rendered on screen, the layout can be considered a 2 column layout with a header and footer.

Layout Structure:

  • body
  • -wrapper
  • -header
  • -container
  • -ltsidebar[hidden]
  • -maincontent
  • -rtsidebar
  • -footer

wrapper:

The wrapper is the "Main Container Div"

It encompasses the entire site layout - header, container and footer. The wrapper has a width of 960px

Purpose:
  • Contains the Layout.
  • Provides Shadow effect across the page.

header: [wrapper>header]

The header provides the Top Level Navigation for the website.

Inherits the Width of the "wrapper" Div, (i.e) 960px. The Top Level Navigation is contained within "masthead" div of the Header.

Header is dynamically generated through JavaScript. Contains the following Script element
<script type="text/javascript">_headerSection();</script>
to generate and display content dynamically

Purpose:

  • Provide Top Level Navigation
  • Provide Website Logo

container[wrapper>container]

container encompasses the 3 Column Layout - leftsidebar, maincontent, rightsidebar. Inherits the width of "Wrapper" Div, (i.e.) 960px, with margin top of 20px.

Visually, divides the maincontent from rtsidebar.

Purpose:
  • Acts as Faux Columns

ltsidebar[wrapper>container>ltsidebar]

Mostly Hidden.

Contains the following Script element,
<script type="text/javascript">_ltsidebarSection();</script>
<script type="text/javascript">_ltsidebarSectionTop();</script>
to dynamically include content or functionality

Purpose:
  • Reserved for future use

maincontent[wrapper>container> maincontent]

maincontent holds the page specific information, (i.e.) the desired content to be served for a user, when they land on a web page. Occupies 680px with a left and radding padding of 20px each.

Contains two Script Elements,
<script type="text/javascript">_maincontentSectionTop();</script>
<script type="text/javascript">_maincontentSectionBottom();</script>
intended to insert content at the beginning of the maincontent div and at the end of the maincontent div.

These script elements can be used not only to add content but also functionality such as use of cookies for page tracking or serving adds dynamically.

As this is the most important part of the layout, providing information to users, it must be organized and structured hierarchically and properly. With regards to CSS and JavaScript, it is necessary to avoid the use of inline styles or scripts, until and unless the situation demands to.

Purpose:
  • renders the page specific Information

rtsidebar[wrapper>container>rtsidebar]

Provides additional information related to the page being viewed. Mostly, used for displaying quick links and adds.

Contains the following Script element,
<script type="text/javascript">_rtsidebarSectionTop();</script>
<script type="text/javascript">_rtsidebarSectionBottom();</script>

Purpose:
  • Providing additional information
  • Cloud Tag

footer[wrapper>footer]

Provides bottom level Navigation, for website related pages, such as the sitemap & archives

Purpose:
  • Bottom Level Navigation

Folders Standards

Folder Organisation

  • /[root]
  • [_css]
  • [_database]
  • [_images]
  • [_script]
  • [Templates]
  • [category-folders]
  • [_css] <optional>
  • [_images]
  • [_script] <optional>

Folder Classification

Folders are classified as RESOURCE folders and CATEGORY folders.
RESOURCE Folders contains resources such as scripts, css, images, database, documents and templates.
CATEGORY Folders contains HTML pages, along with category-specific resource folders.

Folder Names & Contents

  1. "root folder/directory" - contains resource folders[_css, _script, _images, _database, Templates], category folders [about_files, html_files, css_files] and index.html
  2. "_images" folder - contains images[*.jpg, *.jgeg, *.png, *.tiff, *.gif]. Applies to global structure and category structure.
  3. "_script" folder - contains JavaScript or JS Frameworks related files . Applies to global structure and category structure.
  4. "_css" folder - contains Cascading Style Sheets[css]. Applies to global structure and category structure.
  5. "_database" folder - contains site information, page specific information, Database like structure files, documents. Applies only to global structure.
  6. "Templates" folder - contains templates for HTML pages. Note that, it doesn't include a underscore[_], as it related to Dreamweaver. Applies only to global structure.
  7. "Category folder"[Eg: html_files, about-files] - contains HTML,XHTML related files. For index.html, the file should be available in the root dirctory.
    Category folders may or may not contain resource folders.

Folder Naming Convention

  1. Resource folders such as images, css, script, database must begin with Underscore [_]. This is used to indicate priority. Only Category folders will not begin with or contain underscore.
    Eg: _images, _database.
    Irrespective of the location of the resource folders, as to whether they are present globally or local to a specific category, they must begin with a Underscore [_].
  2. Use hypen[-] for spaces. This applies both to Category Folders and Resource Folders. Use of hypen implies, long words be taken as a group of individual words. Long words concatenated by underscore, implies one single word.

File Standards

File Classification

  1. HTML files
  2. CSS files
  3. Script files

HTML Files

  1. html file names or simply URL, can use hypens[-] to denote separation of words.
    Eg: html-tags
    In the above example "html-tags", hypens makes the word to be two individual words [html, tags] rather than a single long word, which is helpful during search.
  2. Bookmarks, which is used for linking to different sections within the same document can contain hypens[-]. underscore is strictly prohibited.
  3. There will be _index<category>.html for every category. This file, will contain all the links to pages available within the respective category.
    Eg: html_files

CSS Files

  1. CSS Files used -
    • reset.css
      - sets default values for the html tags. This is to ensure that the page is rendered consistently across browsers.
    • layout.css
      - contains the page layout styles.
    • component.css
      - contains the style for the various components used in the website.
    • global.css
      - contains most of the styles used across different pages.
    • ie.css
      - applies to IE browser versions starting from IE 7
    • ie6.css
      - applies to IE6. As IE does not have support for CSS Level 2 and higher, this is used to ensure, the pages are rendered consistently in IE6.
  2. Category specific CSS files can be created, if necessary in their respective folders.
  3. Avoid inline styles; Make use of header styling.
    The preferred style of writing would be to write in external css (or) in css file specific to the category.
  4. Provide Commenting for better understandability. Especially for components and layouts.

CSS Naming Convention

  1. ID and class names must be in lowercase and can use hypens[-]. No underscores are allowed.
  2. ID and Class names must be semantic rather than simply being, based on the styling applied.
  3. The name used for an ID, cannot be used for a class and vice-versa.

Script Files

Script forms the base of the website, with the entire Header and Footer being populated using script.
The site is designed to support maximum dynamic changes at any point in time and as required, using client side scripts.
script files used -
  • layout.js
    This script is responsible for populating HEADER, FOOTER Section. Contains functions and variables used for rendering the layout.
    json.js
    Holds the top level navigation and bottom level navigation as JSON Object.
    json_db.js
    JSON Database - Stores information about each page, using Unique Page Identifiers[UPI].
    djs.js
    dynamic javascript framework created for the website.
    common.js
    holds functions and variables used across pages.
    jquery-1.6.2.js
    jquery library
    jquery-effects.js
    contains all the jquery animation and effects used in the website.

Function Naming Convention

  1. Function name will be in camelCase.
  2. Layout based functions, Global function that sets or stores page data, must begin with an Underscore[_]. Other functions can not begin with or contain underscore.
  3. Each function needs to be commented with how-to details and purpose.

Variable Naming Convention

  1. Variable name will be in camelCase. Variable names can contain only Underscore[_]. Hypens are not allowed for script variables.
  2. Global variables must begin with underscore.
  3. Variable must be defined before they are used.

  2. For Every page, ensure json_db, is updated and the data is in sync with each html page data added to the website. Each page must be tested to guarantee the web site's integrity.

  4. Check for Spelling Mistakes.

  7. New pages must be tested thoroughly tested in various browsers.
