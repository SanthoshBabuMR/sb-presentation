When HTML Elements are floated within a Block Element, such as DIV, then the Parent Element, will not CONTAIN the floated Elements or in other words, the height of the parent DIV will be the height of non floating CHILD elements . It is necessary to clear the floating elements within the Parent Element, to avoid any incorrect rendering.

Often, developers stumble upon this issue, when the Parent Element's background-color or background-image doesn't extend the height of its Child elements. Even more annoying is the fact that, this bug seems to appear in FireFox, Chrome etc, while IE tends to render this type of layout as it should.

The reason is, in Interent Explorer, the Child Elements [floating Child or non-floating Child]forces the height of the Parent Element to extend to its height. Thereby, IE gives the wrong impression of rendering properly, while in reality it doesn't.

There are Three Approaches to solving this Issue

Method 1:

Use <div style="clear:both"></div>, after the Floating elements within the Parent Element.

  • Example:Get code for this example
  • <div class="parent_element">
  • <div class="float_left_child">
  • <div>
  • <div class="float_right_child">
  • <div>
  • <div style="clear:both"></div>
  • <div>

Sometimes, additional vertical space appears in IE, due to the use of above markup. This is because, div takes the height of a line of text. Use "height:1px" and "overflow:hidden" to avoid any unwanted gap or vertical space.

Since the above markup is only for presentational purpose, and it is always a better practice to have presentaional markup in CSS, we can further simplify it, by defining the required attribute in a class, and applying it to the div.

  • Example:Get code for this example
  • <style type="text/css">
  • .clrflt{
  • clear: both;
  • height: 1px;
  • overflow: hidden;
  • }
  • <div class="clrflt"></div>

This is the most commonly used approach and the most ideal for cross browser compatability. However, few designers would prefer not to have markup unless it is semantic; for them, the next method could be handy.

Method 2:

Specify height attribute to Parent Element.

  • Example:Get code for this example
  • <style type="text/css">
  • .parent_element{
  • height: 200px;
  • other styles ..
  • }
  • </style>

This is a CSS based approach. By specifying the height attribute, we don't actually clear the floating Child elements, but rather, the visual rendering of the page is made to appear correctly. However, as height of the parent Element is explicitly being provided, it is most applicable to static pages.

If content is dynamic, then it might increase or decrease the actual height of parent Element against the provided height, causing inappropriate rendering. So, we need to constantly check and set the height of the parent DIV to match the height of its enclosing CHILD elements

When developing XHTML pages, where semantic content and standards is given more importance, this could be a preferred approach.

Method 3:

Specify overflow and width attributes to the parent Element.

  • Example:Get code for this example
  • <style type="text/css">
  • .parent_element{
  • overflow: auto/hidden;
  • width: px/em;
  • other styles ..
  • }
  • </style>

This approach makes use of CSS "overflow" and "width" attributes, instead of the "height". The overflow attribute of the parent Element, actually causes the parent Element to wrap its floated children. Moreover, as we don't mention height attribute, it is applicable to dynamic pages and height of the parent Element is adjusted automatically, in accordance with the CHILD elements height.

Note that it is best preferred to give overflow value as "hidden", as this value this rendered correctly across different browsers. The width attribute is required for IE, without which, this rendering can't be obtained.

Each approach has its advantages/disadvantages. The choice of which approach to use, depends on cross browser compatability, the type of content rendered, standards of the page being developed and ofcourse, the user's choice.
